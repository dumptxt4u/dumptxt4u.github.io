    Module 2: Algorithm, Flowchart, and Pseudocode

    Differentiate:

    • Algorithm: Step-by-step procedure to solve a problem.
    • Flowchart: Graphical representation of steps using symbols.
    • Pseudocode: Simplified, language-independent representation of algorithms.

    Example: Simple interest calculation or evaluating an expression (d = a + b * c).

    Definition: A simplified, high-level description of a program or algorithm.

    Reasons for Using Pseudocode:

    • Clarifies the logic before coding.
    • Easy to understand for non-programmers.

    Main Constructs: Sequencing, selection (if-else, case), repetition (loops).

    Key symbols used in flowcharts:

    • Start/End: Oval shape.
    • Arithmetic Calculations: Rectangle.
    • Input/Output Operations: Parallelogram.
    • Decision (Selection): Diamond.
    • For Loop: Hexagon.
    • Flow Lines: Arrows connecting steps.

    Examples to study:

    • Evaluate an expression: d = a + b * c
    • Find simple interest
    • Determine the larger of two numbers
    • Find the smallest of three numbers
    • Determine the grade earned by a student (using if-else and case structures)
    • Print numbers from 1 to 50 in descending order
    • Find the sum of n numbers (using all three loop variants)
    • Calculate the factorial of a number
    • Find the largest of n numbers
